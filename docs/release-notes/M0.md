# M0 — Security Hardening Release Notes

**Release tag:** `v1.0.0-m0`  
**Milestone:** M0 — Security Hardening  
**Authority:** FEB-001 §4 Q1, BHOS-ERR-001 (Conditional Go), ADR-011, ADR-012

---

## Summary

M0 closes critical order API exposure (ERR **CB-1**) and hardens Firestore rules for subscriptions, referrals, and analytics. Delivery uses feature flags for zero-downtime rollout and instant rollback.

---

## What shipped (PR-0 through PR-7 + Release PR)

### API security

| Endpoint | Change |
|----------|--------|
| `GET /api/orders/:id` | Order read access policy; enforced when `FF_ORDER_AUTH_ENFORCE=true` |
| `PATCH /api/orders/:id` | Deprecated; owner/admin required when flag on |
| `POST /api/orders/:id/guest-view-token` | Phone-verified guest JWT issuance (ADR-012) |
| `GET /api/orders/user/:userId` | Firebase auth; self or admin only |
| `POST /api/orders/:id/notify-status` | Firebase auth; tenant owner or admin |
| `POST /api/create-razorpay-order` | Draft ownership bind when `FF_RAZORPAY_DRAFT_BIND=true` |

### Libraries

- `backend-lib/guestOrderToken.ts` — HS256 guest JWT
- `backend-lib/orderAccess.ts` — unified access matrix

### Client

- Guest JWT in `sessionStorage` (`src/lib/guestOrders.ts`)
- Guest order tracking via API (`OrderTracking.tsx`, `OrderSuccess.tsx`, `PaymentSuccess.tsx`)
- Global fetch attaches Firebase Bearer for API calls

### Firestore rules (PR-5)

- Deny `_connection_test_`, `_admin_test_`
- Tighten `subscriptions`, `referrals`, `aiAnalytics`

### CI / tests

- `npm run test:security` — unit + API matrix + rules compile/emulator

## Configuration required (Render)

| Variable | Required | Notes |
|----------|----------|-------|
| `ORDER_GUEST_TOKEN_SECRET` | **Yes** | ≥32 random bytes; server-only |
| `FF_ORDER_AUTH_ENFORCE` | Staging → prod | Default `false`; set `true` after soak |
| `FF_RAZORPAY_DRAFT_BIND` | After validation | Default `false` |

---

## Rollback

| Issue | Action |
|-------|--------|
| Order auth breaks guest/logged-in flows | `FF_ORDER_AUTH_ENFORCE=false` on Render (< 1 min) |
| Razorpay checkout blocked | `FF_RAZORPAY_DRAFT_BIND=false` |
| Firestore permission errors | Redeploy previous `firestore.rules` |

Never delete orders or revoke DB tokens for rollback (guest JWTs are stateless).

---

## Known limitations

- Guest Firestore reads remain denied; guests use API + JWT.
- Referral code lookup still allows any authenticated read on `referrals`.
- Cross-user referrer `subscriptions.pendingDiscount` updates fail until server API exists.
- Full rules emulator tests require JDK 11+ on CI/dev machines.
